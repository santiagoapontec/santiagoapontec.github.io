**3:44 PM** - I just did something with Claude and thought it may be a pretty useful thing to do as a project sometime. Going to write it down here.

I've been working inside a project folder in Claude. I did so that hopefully I could have multiple chats with Claude regarding different aspects of the project but that it would have all the context everytime so that every conversation felt like the same conversation. Meaning that if I spoke to chat #3 of the project I could speak exactly the same way I did to chat #1 and would not have to explain things.

Now, I have one massive chat (apparently its ~5,200 lines of context now) and I just now started a new chat within the project to see if that new window could be where I had Claude help with the more of the conceptual process of understanding to be able to write a notes document. But as soon as I asked Claude if it knew where we were, it said it has no memory and no context (even though that's on in settings). What I did then was simply go to my massive principal chat and literally just copy the entire chat history from front to back, then feed it to the new Claude (as the paste doc it makes when you paste very lengthy things) and boom, it appears to have what seems to me like perfect context.

Now, I assume there's probably some things that could be done to help the chats have even better context, perhaps I don't know, breaking it into whatever the optimal # of context lines per message is, or separating text according to different messages more clearly, or if there's images or docs then being able to attach those and insert little comments referencing image or docs name so that the new window could read that too etc...

Anyways, nothing revolutionary, just a more streamlined and optimized way of doing something that can already be done.

Its just that at least for me that's what I really care about a conversation with an LLM (that's the sticky part of it at least), it's the fact that because we've had such a long interaction I can just talk however I talk and it'll understand - it will get references if I'm using abbreviated words, or it'll understand my explanations or questions even if I ask them badly or don't know how to explain myself because it has context about what we've done, how I think etc...

All of that is lost as soon as I open a new chat window with no memory. And that's especially important for me in projects.

Also the other thing is, for instance Claude allows you to paste memory from other providers, but all it does is ask (prompt) the other LLM for things like; user preferences, important projects etc... sure, that helps, but there's something unexplainable about just having the entire conversation word by word. Saying something like "user has a systemic way of breaking down problems by breaking them down" is a very different thing that seeing that pattern repeated over and over and over with its little subtleties. And that's what I'd like to preserve.

Anyways, cool thing to try and figure out in the future.
